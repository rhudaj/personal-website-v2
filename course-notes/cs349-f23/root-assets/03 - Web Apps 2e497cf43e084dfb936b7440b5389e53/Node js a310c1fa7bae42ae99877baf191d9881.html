npm project files

package.json

  • list of all packages installed in project
  • every npm install adds to this file
  • has information to re-create installed packages
    node init # if package.json exists, installs all packages
  • add to your repo

package-lock.json

  • information to more precisely reproduce /node_modules
  • add to your repo

node_modules/

  • quickly becomes very large
  • important to ignore node_modules/ (put it in .gitignore)
  • can just delete it, then run npm install to re-install all packages

npx (Node Package Execute)

Execute an arbitrary command from an npm package (one installed locally, or fetched remotely), in a similar context as running it via npm exec

ex: 

npx <some-package>
  • If some-package is in your path (it was installed using npm), then it runs the local version of the package
  • If some-package is not in your path (i.e. not installed), then it downloads the latest version of the package and runs it

Vite

Frontend Tooling 2 main parts:

  • A dev server that provides rich feature enhancements over native ES modules
    • TypeScript "transpiling" and source map generation
    • Hot-loading localhost development server
  • A build command that bundles your code with Rollup, preconfigured to output highly optimized static assets for production.

Create a Vite Project

  • has templates
npm create vite@latest # answer Vite project setup questions interactively:
  • npm create initializes a project directory using a "create" package
  • vite in this context means the create-vite package
  • @latest : "use latest version" of create-vite package
npm create vite@latest <project> -- --template vanilla-ts # provide the project name and template directly
