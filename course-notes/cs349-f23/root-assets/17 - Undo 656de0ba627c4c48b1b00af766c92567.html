17 - Undo

we mean undo & redo

Benefits of undo:

  • recover from errors
  • work without fear
  • enables exploratory learning
  • see before/after

Checkpointing

  • simplest form
  • manual undo
    • save copy of the current state to go back later
    • ie: video game checkpoint

Factors

  1. Undoable actions
  1. State: part of UI restored?
  1. Granularity: How much at once time
  1. Scope: global/local

#1 ‚Äî Undoable Actions

  • some actions can‚Äôt be redone
    • eg: window resize, scrollbar position
  • destructible actions:
    • eg: empty trash

Rules of Thumb (relates to MVC)

  • Changes to model/DOM should be undo-able
  • Changes to view/DOM interface state are not undoable
    • unless they change the model (ie: a slider)
  • Ask for confirmation before a destructible action

#2 ‚Äî State Restoration

  • what is UI state after undo/redo
    • eg: highlight text, delete, undo
      • usually highlight text (even though not part of the model)
    • eg: highlight text, delete, scroll, undo
      • usually scroll back to text (even though not part of the model_
  • UI should be meaningful after redo action
    • restore prior selection
    • scroll to show restored
    • give focus to the control where state was restored

#2 ‚Äî Granularity

Chunk: change from 1 state to another (how much to undo)

  • interaction through view divided into undoable chunks
  • undo reverse 1 chunk

undoable chunks: how do we define them?

  • Drawing:

    drawing a line = mousedown + mousemove + mouseup.

    ‚Äúundo‚Äù should undo the entire line.

    chunk = mousedown + mousemove + mouseup

  • Text Editing:

    typed character, word, between commands (bold, delete, etc), between pauses ?

    • VS Code

      chunks = tokens separated by whitespace

    • Word

      chunks = sentence (or up to last deletion)

Tips:

  • ignore direct manipulation in intermediate states (ie: object resize or move states)
  • delimit chunks on discrete input breaks (eg: words in text)
  • Chunk all changes resulting from a single interface event (eg: find and replace multiple words; should undo all replacements at once even though many chunks)

Implementing Undo

2 Approaches

  1. Forward Undo

    * have to compute current document from the start

  1. Reverse Undo
    • save ‚Äúreverse‚Äù commands
    • Undo by applying reverse change to document

Change Record: single change to the document(state of the model)

Implementation of both

Both options need 2 stacks. Commands are moved between them.

  1. undo stack: all change records
  1. redo stack: change records that have been ‚Äúundone‚Äù (so you can reapply them with redo)
  • current state = base value + ‚Äúdo‚Äù commands on the undo stack
    • for forward: regular commands
    • for reverse: reverse commands
  • undo stack ‚Üí redo stack
  • new-state = undo stack
  • Save* = reset the base value + clear undo & redo stacks

Forward (Demo)

  • undo_manager is in the model.
  • instead of storing ‚Äúcount‚Äù for the view the model computes the count from the undo_manager.
  • ‚Äúdo‚Äù commands are increment and decrement.
  • the setter for ‚Äúcount‚Äù is updated to add to the command stack.

Reverse (Demo)

  • redo executes the commands (do)
  • undo executes the reverse commands (undo)
  • model:
    • each mutation method creates a do/undo command
    • undo and redo methods notify observers
  • demo:
    • change slider ‚Äúchunking‚Äù by using ‚Äúinput‚Äù event instead of ‚Äúchange‚Äù 
Command {
	do(): void; 
	undo(): void; 
}

//Example cmd: increment

{ //object in the model 
	do: () => {
	this.count++; 
	}
	undo: () => {
		this.count--; 
	}, 
	as Command
}

Memento (Demo)

  • undo using mementos

    ‚Üí reverse undo, but undo-stack saves snapshots of the state instead of reverse commands

    • ie: for this model, state is just count
    • for more complicated examples, would have to save more parts of the model.
  • undo stack always has current state at top
  • executing undo moves top memento to redo stack, then uses new top of undo stack to set the undo state
    • need at least 2 mementos in the undo stack (1 for the base state, 1 for the current state)
