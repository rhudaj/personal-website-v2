Form Validation

Interfaces often need to¬†validate¬†text input typed by the user.

  • a required field (e.g. credit card number)
  • a certain format (e.g. postal code)¬†
  • within a certain range
  • unique (e.g. choose a username)

Why?

The model expects certain kinds of data, or logic won't work

  • get the right data at the interface level
  • deliver the right data to the model

To guide users

Protect the system

  • attacks mounted through unprotected text submission

Guidelines

  • Place error messages near fields
  • Use colour to differentiate errors from normal field states¬†
  • When possible
    • accept data formatted in different ways
    • filter invalid characters from being entered¬†
    • validate a field before input is complete
